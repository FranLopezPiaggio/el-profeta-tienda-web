## Usage Guidelines

### When to use Tailwind classes vs CSS variables
- **Default: Use Tailwind classes** (e.g., `text-gray-900`) — Tailwind's default palette already maps to our design tokens
- **CSS variables: Only when dynamic** — Use `text-[var(--color-gray-900)]` for dynamic theming or hover states computed in JS
- **Rationale:** Tailwind v4's `@theme` directive and default color palette are pre-configured to match our brand colors

### Color hierarchy
1. **Primary (`gray-900`, `#111827`)**: Headings, buttons, key UI elements
2. **Secondary (`gray-600`, `#4B5563`)**: Body text, nav links
3. **Muted (`gray-400`, `#9CA3AF`)**: Placeholders, disabled text
4. **Border (`gray-200`, `#E5E7EB`)**: Dividers, card borders
5. **Background (`white`, `#FFFFFF`)**: Page background

## Future: Adding New Components

When adding new components, use the color hierarchy above:

```tsx
// Primary CTA button
<button className="bg-gray-900 text-white hover:bg-gray-800">
  Call to action
</button>

// Secondary/muted text
<p className="text-gray-600">
  Supporting information
</p>

// Card or panel
<div className="bg-white border border-gray-200 rounded-lg">
  Content
</div>
```

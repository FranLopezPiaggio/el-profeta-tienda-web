**Replace the entire content with:**
```css
@import "tailwindcss";
@import "../design-system/tokens.css";

@theme {
  --font-sans: var(--font-montserrat), ui-sans, system-ui, sans-serif;
  --font-serif: var(--font-playfair), ui-serif, Georgia, serif;
}

/* Antares Design System - Root-level token exposure
 * Makes design tokens available as Tailwind arbitrary values
 * Usage in components: text-[var(--color-gray-900)]
 * Usage in arbitrary: bg-[var(--color-white)]
 * This is optional - Tailwind's built-in color names (gray-900, etc.)
 * are already mapped to the correct values via Tailwind's default palette.
 * The tokens.css file provides the canonical reference.
 */
:root {
  --color-gray-900: #111827;
  --color-gray-600: #4B5563;
  --color-gray-500: #6B7280;
  --color-gray-400: #9CA3AF;
  --color-gray-300: #D1D5DB;
  --color-gray-200: #E5E7EB;
  --color-gray-100: #F3F4F6;
  --color-gray-50: #F9FAFB;
  --color-white: #FFFFFF;
}
```

**Important notes:**
- `@import "../design-system/tokens.css"` — path is relative from `app/` to `design-system/`
- `:root {}` block makes tokens available globally as CSS variables
- Tailwind arbitrary value syntax allows using these: `text-[var(--color-gray-900)]`
- Most components will continue using Tailwind's built-in color names (e.g., `text-gray-900`) because Tailwind's default palette already matches the design tokens
- The `:root {}` block enables using CSS variables when needed (e.g., for dynamic theming)
